# VisionX — Master Plan Βελτιώσεων (Ιούλιος 2026)

Στόχος: κάθε όχημα και άτομο να εντοπίζεται **μία φορά**, να ακολουθείται σε όλη
την πορεία του χωρίς απώλειες, με τα καθαρότερα στιγμιότυπά του, την πινακίδα του
(αν είναι όχημα), καθαρή λήψη προσώπου (αν είναι άτομο), και αναφορά που δείχνει
χρονικά διαστήματα εμφάνισης με gallery. Πλατφόρμα αναφοράς: **NVIDIA laptop**
(με διατήρηση συμβατότητας Mac όπου δεν κοστίζει).

Θεμελιώδης αρχιτεκτονική απόφαση: το VisionX είναι **offline** εργαλείο — δεν
χρειαζόμαστε τέλειο real-time tracking. Κρατάμε το BoT-SORT για το πρώτο πέρασμα
και προσθέτουμε **δεύτερο πέρασμα συρραφής (tracklet stitching)** που διορθώνει
εκ των υστέρων ό,τι έσπασε: η προσέγγιση των νικητών των tracking challenges
2025-2026 (GTA: hierarchical clustering των tracklets με ReID appearance
embeddings + χωροχρονικούς περιορισμούς).

Κάθε φάση είναι αυτοτελώς παραδοτέα και επαληθεύεται σε πραγματικό υλικό
(μέθοδος ground-truth όπως στη βαθμονόμηση του plate.py).

---

## Φάση Α — Αδιάσπαστο tracking (η καρδιά του πλάνου)

**A1. Best-K στιγμιότυπα ανά track.** Κατά την επεξεργασία, κράτηση των K
καθαρότερων crops ανά track (ευκρίνεια Laplacian × μέγεθος × conf ανίχνευσης —
επαναχρησιμοποίηση της λογικής του plate.py). Αποθήκευση για report gallery.

**A2. Offline tracklet stitching (δεύτερο πέρασμα).** Μετά το BoT-SORT:
- Εξαγωγή appearance embedding ανά tracklet (μέσος όρος στα best crops).
  Πρώτη δοκιμή: **CLIP** (υπάρχει ήδη στο project για το YOLOE — μηδέν νέα
  dependencies)· εναλλακτικά OSNet (ελαφρύ, 2.5M, μέσω BoxMOT/torchreid ONNX).
- Hierarchical clustering (cosine distance μέσων διανυσμάτων) με πύλες: ίδια
  κλάση, μη επικαλυπτόμενα χρονικά, συμβατή θέση/κίνηση στα άκρα.
- Συγχώνευση συστάδας σε μία ταυτότητα με **λίστα χρονικών διαστημάτων**.
- Για οχήματα: η πινακίδα (Φάση Β) ως ισχυρό κλειδί συγχώνευσης.
- Μετρική αποδοχής: σε δοκιμαστικά κλιπ με γνωστό πλήθος αντικειμένων, τα
  διπλά entries στην αναφορά να πέφτουν δραστικά χωρίς λάθος συγχωνεύσεις.

**A3. Σταθμευμένα/στατικά αντικείμενα.** Κατάσταση track (κινούμενο/στατικό)
από διακύμανση θέσης. Στατικό αντικείμενο = «άγκυρα»: οι προσωρινές αποκρύψεις
από διερχόμενα ΔΕΝ γεννούν νέο ID (επανασύνδεση με θέση+εμφάνιση στο stitching)·
στην αναφορά μία εγγραφή «σταθμευμένο 12:01-14:33», όχι εκατοντάδες.

**A4. Fine-tuning tracker.** track_buffer/όρια ανά περίπτωση χρήσης, επιβεβαίωση
with_reid, προαιρετικό A/B με OccluBoost (BoxMOT) αργότερα — μόνο με μετρήσεις.

## Φάση Β — Αυτόματες πινακίδες στην κύρια αναφορά

**B1. Refactor του plate.py πυρήνα** σε εισαγώγιμο module (plate_core):
detector, OCR ensemble, ψηφοφορία, χωρίς GUI.

**B2. Ενσωμάτωση στο vision pipeline:** για κάθε track οχήματος, plate detection
στα best-K crops → OCR + ψηφοφορία → κορυφαίος υποψήφιος + confidence στο track.
(Σχεδίαση ήδη επικυρωμένη: two-stage στα υπάρχοντα crops, gated ανά κλάση.)

**B3. Στην αναφορά:** chip πινακίδας στην κάρτα οχήματος + κουμπί «Βαθιά ανάλυση
πινακίδας» που προσυμπληρώνει ROI/καρέ και καλεί τη ροή του plate.py.

**B4. Plate-only mode:** επιλογή στο GUI «μόνο εξαγωγή πινακίδας» για ένα βίντεο
(η υπάρχουσα ροή plate.py μέσα από την εφαρμογή).

## Φάση Γ — Πρόσωπα (best-shot εξαγωγή, ΟΧΙ αναγνώριση)

**C1.** Face detection με **SCRFD** (ONNX, ταχύτητα με ακρίβεια RetinaFace) στα
best crops των person tracks. Βαθμολόγηση ποιότητας (μέγεθος, ευκρίνεια,
μετωπικότητα από landmarks) → 1-3 καλύτερες λήψεις προσώπου ανά άτομο στην
αναφορά. Ρητά ΕΚΤΟΣ scope: face recognition/ταυτοποίηση — μόνο εξαγωγή
καθαρού στιγμιότυπου για ανθρώπινη εξέταση.

## Φάση Δ — Αναφορά v2

**D1.** Κάρτες αντικειμένων: κλάση, διαστήματα εμφάνισης (μετά τη συγχώνευση),
κουμπί gallery με τα best στιγμιότυπα (lightbox), chip πινακίδας, chip προσώπου,
φίλτρα/ταξινόμηση. Στατικά οχήματα ως συμπτυγμένες εγγραφές.

**D2.** GUI: ενσωμάτωση plate mode (B4) + μόνιμες ρυθμίσεις + πεδίο φακέλου
εξόδου + έκθεση stride/parallel.

## Φάση Ε — Πολλαπλές κάμερες / ίδιο συμβάν

**E1.** Offline αντιστοίχιση μεταξύ βίντεο: (α) ταύτιση πινακίδων (ακριβής +
fuzzy στα per-char εναλλακτικά) ως ισχυρό κλειδί για οχήματα, (β) appearance
embeddings (CLIP/CLIP-ReID) + χρονικά παράθυρα για άτομα και οχήματα χωρίς
αναγνώσιμη πινακίδα. Συγκεντρωτική αναφορά: «Αντικείμενο Χ: Κάμερα Α
12:01-12:03 → Κάμερα Β 12:05-12:07», με συνδυασμό ψήφων πινακίδας από όλα τα
περάσματα (καλύτερη ανάγνωση από πολλαπλές οπτικές).

**E2.** Ενοποίηση με το υπάρχον chain mode.

## Φάση ΣΤ — Υγιεινή (παράλληλα με τις άλλες)

Stage 0 bugs (επαλήθευση + διόρθωση), Python tests για voting/fusion/stitching,
benchmark closed-set YOLO26 vs YOLOE-26 στις 6 κλάσεις-στόχους σε πραγματικό
υλικό (πιθανή απλοποίηση + επιτάχυνση), αξιολόγηση SVTRv2 ως 4ο μέλος OCR
ensemble.

---

## Προτεινόμενη σειρά υλοποίησης

**Α → Β → Δ → Γ → Ε** (με ΣΤ παρεμβαλλόμενη). Η Α λύνει το πιο ενοχλητικό
πρόβλημα (διπλές εγγραφές), η Β δίνει τη μεγαλύτερη νέα αξία (πινακίδες παντού),
η Δ τα κάνει ορατά στον χρήστη, η Γ προσθέτει τα πρόσωπα, η Ε είναι η πιο
φιλόδοξη και πατάει πάνω σε όλα τα προηγούμενα.

## Τεχνικές αναφορές

- Tracklet stitching: GTA (arXiv 2411.08216), GTATrack SoccerTrack 2025 winner
  (arXiv 2602.00484) — hierarchical clustering ReID embeddings.
- ReID: OSNet (ελαφρύ, in-domain) vs CLIP-ReID (καλύτερο cross-camera) —
  arXiv 2601.20598 σύγκριση 2025.
- Faces: SCRFD (InsightFace) + best-shot quality selection (καθιερωμένη
  πρακτική εμπορικών συστημάτων).
- Στατικά: dual-background + occlusion-tolerant association (βιβλιογραφία
  video analytics).
- Πλήρης τεκμηρίωση έρευνας Ιουλίου 2026: βλ. μνήμη project
  (reference_sota_research_2026_07).
